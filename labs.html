  2. Copy-paste this content into the file.

    #!/bin/bash
# This is a 15-minute countdown timer.

end_time=$((SECONDS+900))  # Set the end time to 15 minutes (900 seconds) from now

while [ $SECONDS -lt $end_time ]; do
    remaining_seconds=$((end_time - SECONDS))
    echo "Time remaining: $remaining_seconds seconds" > /dev/null
    sleep 1
done

echo "Countdown complete! Ring..."

  3. Save the file and exit nano.

    ./timer.sh &

Note: A process of timer.sh is created each time you execute this command. The & symbol enables timer.sh to run in the background.

In this lab, we’ll set up a Local Area Network (LAN) using a router and up to 10 devices. This Local Area Network is similar to the setup you use at home.

  1. Power up the router.

  2. Connect one laptop to the router using an Ethernet cable or WiFi.

  3. Access the router’s configuration page: open a web browser and enter the router’s IP address in the address bar. Log in to the router using the default username and password.

  4. Change the password to something more secure. Each team needs to know their own password, but do not share it across teams

  5. Configure the router’s Service Set Identifier (SSID) and Pre-Shared Key (PSK) (ie. passphrase).

  6. Configure the router’s IP address to be 192.168.T.1, where T == the team’s number.

  7. Configure the router’s DHCP server to assign IP addresses 192.168.T.10 to 192.168.T.100, where T == the team’s number. Use subnet mask 255.255.255.0.

  8. Ensure that all devices can ping the router and ping each other. It may be necessary to disable Windows Firewall temporarily (remember to enable it after the lesson).

  9. Run a web server on one of the laptops (in a non-sensitive folder). Ensure other team members can connect to that web server. On Windows, use Windows Terminal or similar, not WSL.

  10. Draw the network topology diagram for the network. include

    1. Devices: phones, laptop, router

    2. Connections between devices: Ethernet, WiFi

    3. Hostnames and IP addresses, subnet masks, and gateway addresses of each device

6.2.1. HTTP Server on AWS EC2

  • Amazon Web Services (AWS) is a cloud service provider. In this lab, we’ll learn how to start an AWS instance (a virtual computer) on the cloud, to serve web pages

Login to the AWS Learner Lab, open the AWS Management Console

  1. Retrieve the AWS invitation email you received. Copy the email address mentioned in the invitation email used for account creation.

  2. Go to https://www.awsacademy.com/vforcesite/LMS_Login

  3. Choose Student Login ➜ Login using your e-mail address (use ONLY the email address mentioned in the invitation email) and password used for account creation.

  4. Go to Courses ➜ AWS Academy Learner Lab [#####] ➜ Modules ➜ Learner Lab ➜ Start Lab

  5. Wait for AWS🔴 to become AWS🟢

  6. Click on AWS🟢. This launches the AWS Management Console

Launch an EC2 instance

An Elastic Compute 2 (EC2) instance is like a computer somewhere on the cloud that you can use.

  1. AWS Management Console ➜ Search ➜ EC2 ➜ Instances ➜ Launch an instance

  2. Name and tags: http-server

  3. Application and OS Images: Ubuntu

  4. Instance type: t2.micro

  5. Key pair: Create new key pair

    1. Key pair name: is114-http-keypair

    2. Key pair type: RSA

    3. Private key file format: .pem

    4. Create key pair

    5. A file named is114-http-keypair.pem will be saved to your computer. This is your private key for accessing the server. Remember where you saved this file, it’s needed later.

  6. Network settings:

    1. Allow SSH traffic from the internet

    2. Allow HTTPS traffic from the internet

    3. Allow HTTP traffic from the internet

  7. Leave other settings as default

  8. Click Launch instance

  9. Wait for instance to be launched

  10. Click View all instances

  11. Look for the Public IPv4 Address of the instance, copy it somewhere. When you see YOUR_EC2_INSTANCE_IP_ADDRESS below, insert this address

Launch a CloudShell instance

AWS CloudShell is a browser-based shell. You can think of it as yet another computer. It is not the same as the EC2 instance launched previously.

  1. Start AWS CloudShell

  2. Notice that the prompt is cloudshell-user@SOMETHING. This means you’re logged into the CloudShell instance.

  3. Actions ➜ Upload file ➜ Upload your private key, is114-http-keypair.pem. Note: usually, you would keep the private key on only one computer, and not copy it to other computers. In this case, it’s for practice only and it’s fine.

  4. Change the permissions of your private key to prevent people from viewing it:

    chmod 600 is114-http-keypair.pem

SSH from the CloudShell instance to the EC2 instance

  1. SSH from the CloudShell instance into the EC2 instance:

    ssh -i is114-http-keypair.pem ubuntu@YOUR_EC2_INSTANCE_IP_ADDRESS

  2. The prompt changes to ubuntu@ip-SOMETHING. This means you are logged in to the EC2 instance.

  3. Check your Public IP address. It should be the same as YOUR_EC2_INSTANCE_IP_ADDRESS:

    curl checkip.amazonaws.com

Update software on the EC2 instance

  1. Are you logged into the EC2 instance (check the prompt)? If not, make sure that you are.

  2. Update the EC2 instance software:

    sudo apt update
sudo apt upgrade

  3. Reboot:

    sudo reboot

  4. After rebooting, the prompt changes to cloudshell-user@SOMETHING, this means you are not logged in to the server anymore, you’re back at the CloudShell instance

  5. Wait for a few minutes for the EC2 instance to reboot

    #!/usr/bin/env python3
# -*- coding: utf-8 -*-

import dweepy
import requests
import pprint

"""
Don't need to understand all the code below, only the relevant parts
"""
def main() -> None:
    # the thing name to get dweets from
    thing_name = "CHANGE_THIS_TO_SOMETHING_UNIQUE"

    # go into a loop, waiting for dweets
    print()
    while True:
        # Use try except, cos of https://github.com/paddycarey/dweepy/issues/14
        try:
            for dweet in dweepy.listen_for_dweets_from(thing_name):
                print(f"dweet received:")
                pprint.pprint(dweet)
        except requests.exceptions.ConnectionError as e:
            print(f"dweet.io closed the connection, reconnecting")

if __name__ == "__main__":
    main()
